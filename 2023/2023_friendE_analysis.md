상위 항목: 2023년 심의 결과 분석 (https://www.notion.so/2023-2a566d0370c680219042f9a91b2ffb7e?pvs=21)

# 1) 2023년 심의자료 → 기존 8가지 평가기준 매핑(핵심만)

- **이차자료 의존 / 출처 미표기·바이라인 오용**
    
    타 매체(통신사/뉴스1·연합뉴스 등) 기사를 거의 그대로 전재하면서 **자사 바이라인을 부착**하거나 출처를 누락. → *1-2. 이차 자료 의존*과 *2. 투명성과 책임성(취재원·출처 투명성)*에 해당.
    
    (기준 근거 예시: “이차 자료 의존/인용 출처 미표기”, “기자·매체 정체성 불투명”.)
    
- **기사–광고 구분 불명확(네이티브/위장 광고)**
    
    매체 홈페이지 UI에서 기사 목록과 광고(배너·박스)를 **같은 묶음**에 배치하거나 ‘기사더보기▶’ 등 기사로 오인되는 문구로 광고 클릭 유도. → *4-2. 상업적 이익 우선/광고성 기사* + *2. 투명성과 책임성*.
    
- **낙인·비하성 제목(집단 대상 오명화)**
    
    최저임금 보도에서 ‘**민노총의 몽니**’와 같이 특정 집단을 비하·오명화하는 표현을 제목에 사용. → *7. 언어와 표현의 윤리(비속어·혐오/차별 표현, 과장·왜곡 제목)*에 해당.
    
- **관계(친족·연고)로 인물 식별·클릭 유도**
    
    사건 당사자를 본인 실명 대신 ‘**태진아 아들**’ 등 **유명인과의 관계로 호명**하는 제목 다수. → *1-5. 인권·프라이버시 존중*과 *7. 언어와 표현의 윤리(과장/선정적 제목)* 보강 필요.
    
- **보도자료·특정 기획(별지 섹션) 기반의 사실상 홍보성 패키지**
    
    ‘부울경 관광명소’ 등 **별지 섹션 형태**로 편성되어 공익적 보도와 광고/홍보의 경계가 흐려짐(보도자료 검증 문제와 결합). → *1-2. 보도자료 저널리즘* + *4-2. 상업적 이익 우선*에 해당하며 표기·공개 강화 필요.

# 2) 기존 기준에 없는 “새로운/특별한 패턴” 제안(2023 근거 중심)

아래 표는 **신규 패턴 정의–왜 문제인지–어느 기준에 넣을지(추가 위치)**를 한눈에 보시도록 정리한 것입니다.

| 새로 발견/명시 필요 패턴 | 핵심 정의(운영상 포인트) | 기존 기준에의 편입 제안(정확한 위치) | 2023 근거 사례 |
| --- | --- | --- | --- |
| **UI/레이아웃 기반 위장 광고** | 홈·앱 인덱스/박스에서 기사·광고를 동일 영역·동일 톤으로 배열하거나 ‘기사더보기▶’ 등 기사로 오인 유도. **레이블·시각적 구분 의무**를 체크리스트에 포함. | *4-2. 상업적 이익 우선* 하위에 **“UI 기반 기사–광고 오인 금지”** 신설 + *2. 투명성*에 **“광고 레이블 가이드”** 보강 | 부산일보 ‘실시간 핫뉴스’ 블록 상단 광고 혼치 배열(기사·광고 구분 불명확) 심의. |
| **관계 호명(친족·연고) 헤드라인** | 당사자 실명을 두고 **유명인과의 관계로 호명**해 클릭을 유도하거나 낙인을 강화. 사건 관련성이 명백할 때만 제한적 허용, **불가피 시 본문에서 맥락·필요성 설명**. | *1-5. 인권·프라이버시* 하위에 **“관계(친족·연고) 호명 제한”** 신설. *7-1. 과장/왜곡 제목*에 **예시 추가** | “태진아 **아들** 이루 …”를 일제히 제목 전면에 배치한 다수 매체. |
| **낙인어·감정 단정어 사전(편집금지/주의 리스트)** | ‘**몽니**’ 등 집단을 비하·오명화하는 단어, 감정적 단정어를 **제목 금칙/주의어**로 관리. 인용 필요 시 **따옴표+맥락/출처·반론** 병기. | *7-3. 비속어·혐오 표현*에 **“낙인어 리스트 & 사용 기준”** 조항 추가. *10조(편집지침·제목)* 준수 체크 | ‘민노총의 **몽니**’ 등 집단 비하성 제목 제재. |
| **전재 기사에 자사 바이라인 부착(바이라인 오용)** | 타사/통신사 기사 **문장 수준 전재**에 자사 바이라인 부착. **원문 링크·출처·가공 범위 공지**를 의무화, 자동 점검 로직(유사도) 제안. | *2-3. 기자·매체 정체성 불투명*에 **“바이라인 오용 금지”** 명시 강화 + *1-2. 인용 출처 미표기* 연동 | 뉴스1·연합뉴스 기사 전재에 자사 바이라인 부착 사례 다수. |
| **별지/기획 섹션 기반 네이티브 패키지** | **별지·특집 섹션**에 광고/보도자료성 콘텐츠를 **연재형 패키지**로 편집. 섹션 상단에 **제작 주체·재원·검증수준** 등 **레이블 의무**. | *4-2. 상업적 이익 우선*에 **“섹션·기획 단위 표시의무”** 신설. *2. 투명성*에 **“패키지형 콘텐츠 공시”** | ‘부울경 관광명소’ 별지 섹션 및 보도자료 검증 문제 제기. |
| **제목·본문 간 텍스트 재가공형 전재(“표절-경계” 회피)** | 원문을 **부분 삭제·토씨 수정**만으로 재가공해 **자체 기사처럼 유통**. **“실질 동일성+가공 범위 공개”** 기준 마련. | *1-2. 이차 자료 의존* 하위에 **“경미 가공 전재의 공개 의무”** 추가 | 타사 기사 일부 문장만 수정·삭제 후 전재, 출처 미표기 사례. |

> 위 6가지 항목은 기존 기준의 취지와 충돌하지 않으면서도 2023년 심의 사례에 비춰 운영·점검 가능성을 높이도록 설계했습니다.

# 3) 기준 문구(운영 지침) 보강 제안

아래는 CR 템플릿/가이드에 바로 반영할 수 있는 **짧은 문구**들입니다. (각 문구는 해당 기준 하위에 추가)

- **[2. 투명성과 책임성] 바이라인 오용 금지(신설)**
    
    “타 매체/통신사 보도를 인용·전재하는 경우, **원문 링크·출처·가공 범위(편집·요약·해설 여부)**를 기사 서두 메타영역에 명시한다. 원문 문장 유사도가 높을수록 자사 바이라인 단독 표기는 금지한다.” (사례 근거: 전재+자사 바이라인 부착 제재)
    
- **[4-2. 상업적 이익 우선] UI 기반 기사–광고 오인 금지(신설)**
    
    “홈/앱 인덱스·박스·모듈에 기사와 광고를 혼치 배열하지 않는다. 광고는 **AD/스폰서**를 굵은 라벨로, **시각적 톤과 인터랙션(버튼·호출문구)**도 기사와 명확히 구분한다.” (사례 근거: ‘실시간 핫뉴스’ 혼치 배열)
    
- **[1-5. 인권·프라이버시] 관계 호명 제한(신설)**
    
    “사건과 무관한 **친족·연고 호명(‘○○의 아들/부인’)을 제목에서 금지**한다. 공익상 불가피할 때는 **본문에서 맥락·필요성**을 설명한다.” (사례 근거: ‘태진아 아들’ 대량 제목 제재)
    
- **[7-3. 비속어·혐오 표현] 낙인어 리스트 운용(보강)**
    
    “집단 비하·오명화 단어(예: **‘몽니’**)를 **제목 금칙/주의어**로 관리한다. 직접 인용은 따옴표 처리하고 **출처·맥락·반론**을 병기한다.” (사례 근거: ‘민노총의 몽니’ 제목 제재)
    
- **[1-2. 이차 자료 의존] 경미 가공 전재 공개(보강)**
    
    “타 기사 **부분 삭제·토씨 수정** 수준의 재가공은 **‘전재/요약’ 고지**와 **원문 링크**를 의무화한다.” (사례 근거: 부분 발췌·토씨 수정 전재 제재)
    
- **[4-2. 상업적 이익 우선] 패키지형(별지·기획) 공시(신설)**
    
    “별지/기획 섹션은 **제작 주체·재원·검증 수준(보도자료 기반 여부)**를 섹션 헤더에 고정 표기한다.” (사례 근거: 별지 섹션·보도자료 검증 문제)

# 4) 실무 적용 팁(템플릿 자판기 반영 포인트)

- **자동 점검 훅**
    
    ① 기사본문–외부 원문 **유사도 경고(바이라인 오용 방지)** → 출처·링크·가공범위 입력란 강제 노출. (근거: 전재+바이라인 제재)
    
    ② **헤드라인 필터**: 금칙·주의어 사전(‘몽니’ 등) 통과 시, 대체 표현 추천 및 **반론·맥락 병기 체크리스트** 자동 표출. (근거: 집단 비하성 제목 제재)
    
    ③ **광고 레이블 검사**: 홈/앱 모듈에서 기사·광고가 같은 박스에 들어가면 경고. **AD 라벨·색/아이콘 구분** 체크. (근거: 기사–광고 혼치 배열 제재)
    
- **리포트 문구 프리셋(간단 예시)**
    
    “**문제적 패턴: UI 기반 기사–광고 오인** — 귀 매체 홈 상 **[모듈명]**은 기사와 광고가 동일 톤으로 배열되어 독자 오인을 유발합니다. **AD 라벨·색분리·호출문구 수정**이 필요합니다. (윤리근거: 기사–광고 구분 의무)”
